## 🔴 严重问题 (P0 — 阻断核心功能)

### 1. WebSocket 未连接 — Agent 实时对话不可用

- **位置**: 所有页面右下角 Agent 面板
- **现象**: 状态显示 `● 未连接`，Token: 0
- **影响**: **AI Agent 对话是核心功能**，未连接意味着用户无法与 AI 交互，整个 Agent 协作流程瘫痪
- **证据截图**: `bug1_script_step.png`, `bug4_characters.png` 等所有截图

### 2. API 路由不一致 — `/api/projects` 返回 404

- **现象**: 前端调用 `/api/projects` 返回 404，但 `/api/v1/projects` 正常返回数据
- **影响**: 前端可能无法正确加载项目列表（当前靠 RSC/SSR 绕过了）
- **控制台**: 14 个 404 错误持续输出

### 3. 剧本步骤 — "质量评估" Tab 严重截断

- **位置**: ScriptMind → 步骤5 剧本内容
- **现象**: `质量评估` 三个字被竖排截断显示为 `质\n量\n评\n估`，与 `剧本内容`/`角色`/`场景/布景` 横排Tab 不一致
- **影响**: UI 严重破损，用户无法正常点击切换
- **证据截图**: `bug1_script_step.png`

### 4. 移动端 ScriptMind 步骤名称被截断

- **位置**: 375px 宽度下的 ScriptMind 页面
- **现象**: 五个步骤仅显示数字 `1 2 3`，步骤名称（创意及世界观、梗概、角色等）完全不可见
- **影响**: 移动端用户无法知道自己在哪个步骤，工作流导航基本失效
- **证据截图**: `bug10_mobile_scriptmind.png`

## 🟠 重要问题 (P1 — 影响用户体验)

### 5. 项目详情页 — 无明确进入 ScriptMind 的入口

- **位置**: `/projects/[id]` 项目工作台页
- **现象**: 页面显示"开始创作你的剧本"引导文案和 Agent 聊天面板，但**没有明显的按钮**跳转到 `/projects/[id]/scriptmind`
- **影响**: 用户不知道如何进入核心的五步骤工作流

### 6. 世界观表单 — 无必填字段标记

- **位置**: ScriptMind → 步骤1 创意及世界观
- **现象**: `题材类型`、`风格基调` 等字段没有 `*` 必填标记，但提交时可能需要
- **影响**: 用户不知道哪些字段必须填写，可能导致保存失败却无明确提示

### 7. 保存操作 — 无成功反馈

- **位置**: 世界观、梗概等步骤的保存按钮
- **现象**: 点击保存后无 toast/snackbar 提示"保存成功"
- **影响**: 用户不确定数据是否已保存

### 8. 设置页面 — MCP Server / 其他工具 标签页内容异常

- **位置**: `/settings` 设置页
- **现象**: 点击 `MCP Server` 和 `其他工具` 标签后页面内容无变化（仍显示模型供应商列表）
- **影响**: 这两个配置功能不可用

### 9. 大纲类型切换 — UI 表现不一致

- **位置**: ScriptMind → 步骤4 大纲
- **现象**: 测试脚本中 `text=集数列表, text=幕次结构` 匹配为 0（实际是 dropdown 选择器），但与之前版本的 tab 切换不一致
- **影响**: 用户可能困惑于大纲类型的切换方式

## 🟡 一般问题 (P2 — 改进建议)

### 10. 项目卡片 — 点击行为不直观

- **现象**: 项目卡片是 `<div class="cursor-pointer">` 而非 `<a>` 链接，无法右键新标签打开
- **影响**: 不支持 Ctrl+Click 新标签打开，不符合 Web 习惯

### 11. Agent 角色按钮 — 排列拥挤

- **位置**: ScriptMind 右侧 Agent 面板顶部
- **现象**: `主笔编剧`、`设定架构师`、`角色设计师`、`剧本医生` 四个按钮在窄面板中被截断
- **影响**: 部分角色名不可见

### 12. Token 计数器 — 始终显示 0

- **位置**: Agent 面板右下角
- **现象**: 无论是否使用 AI，Token 始终为 0
- **影响**: 用户无法了解 AI 消耗情况

### 13. 项目侧边栏 — 模块功能未实现

- **位置**: 项目详情页左侧 `分镜`、`风格`、`动态`、`配音` 模块
- **现象**: 这些模块按钮存在但点击后无明显功能变化
- **影响**: 用户期望这些模块有对应功能，但实际为空壳

## 结论

应用的**基础框架和 UI 骨架**已经搭建完成，五步骤工作流路由正确，表单字段齐全。但**核心 AI 功能（WebSocket 连接）完全不可用**，这是最严重的问题。此外有多处 UI 布局破损和交互反馈缺失，需要优先修复。

**总结**: 3 个 P0 严重问题 + 5 个 P1 重要问题 + 5 个 P2 改进建议。建议优先解决 P0 问题后再进行功能验收。
