# Feature Specification: ScriptMind 前端重建

**Feature Branch**: `002-scriptmind-frontend`

**Created**: 2026-06-18

**Status**: Draft

**Input**: 基于 001-scriptmind-screenplay-factory 的完整 spec，重新设计前端，彻底清除现有前端代码后从零重建

## Clarifications

### Session 2026-06-18

- Q: 前端重建的范围是什么？→ A: 仅前端 (`frontend/`) 目录，后端 API 契约不变，完全参照 001 spec 的 API Contracts 和 Data Model。
- Q: 前端技术栈是否需要调整？→ A: 保持 Next.js 14 + shadcn/ui + Tailwind CSS + Zustand + Slate.js，但需重新组织代码结构，清除所有遗留代码。
- Q: 前端是否需要支持多语言？→ A: 初始版本仅支持中文界面，不做 i18n。
- Q: 编辑器的自动保存和版本快照策略是什么？→ A: 30秒防抖自动保存（防丢数据），仅手动保存时创建版本快照。
- Q: 后端服务完全不可用时，前端应如何处理？→ A: 显示全屏错误页，提供"重试连接"按钮，已编辑的草稿保留在本地存储中，后端恢复后可继续。
- Q: 删除项目时的确认方式和数据处理策略是什么？→ A: 二次确认对话框，要求用户输入项目名称确认删除，级联删除所有关联数据（剧本、角色、伏笔等）。
- Q: Agent 生成过程中用户导航离开当前页面时，系统应如何处理？→ A: 后台继续执行，导航栏显示小型进度指示器（旋转图标 + 阶段名称），用户可随时返回查看。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 首页与项目导航 (Priority: P1)

用户访问首页，看到平台介绍和工作流程说明，点击"打开工作台"进入项目列表页。项目列表页展示所有已创建项目，支持新建项目。点击某个项目进入项目工作台。

**Why this priority**: 首页和项目导航是用户进入系统的第一个触点，必须流畅且直观。没有项目列表和创建功能，后续所有功能无法启动。

**Independent Test**: 用户访问首页，点击"打开工作台"，在项目列表页点击"新建项目"，填写标题和题材后创建成功，项目出现在列表中。

**Acceptance Scenarios**:

1. **Given** 用户访问首页，**When** 页面加载完成，**Then** 显示平台介绍、工作流程说明、Agent 团队介绍和核心能力描述
2. **Given** 用户在首页点击"打开工作台"，**When** 跳转完成，**Then** 显示项目列表页，列出所有已创建项目（按更新时间倒序）
3. **Given** 用户在项目列表页点击"新建项目"，**When** 填写标题、选择题材标签和目标形态后提交，**Then** 新项目创建成功并跳转到该项目工作台
4. **Given** 项目列表为空，**When** 用户访问项目列表页，**Then** 显示空状态引导，提示用户创建第一个项目
5. **Given** 用户在项目列表页点击某个项目卡片，**When** 跳转完成，**Then** 进入该项目的工作台页面，显示项目详情和模块入口
6. **Given** 用户在项目列表页点击某个项目的"删除"按钮，**When** 系统弹出二次确认对话框要求输入项目名称，**When** 用户正确输入项目名称并确认，**Then** 项目及其所有关联数据（剧本、角色、伏笔等）被删除，项目从列表中移除

### User Story 2 — 一句话创意生成完整大纲 (Priority: P1)

用户在项目工作台的 Agent 交互面板中输入一句话梗概（如"写一个现代都市复仇短剧，女主被渣男背叛后逆袭"），选择风格预设，系统通过 AI Agent 协作自动生成结构化大纲。界面实时显示 Agent 阶段进度和流式输出内容。生成完成后进入人类审核节点，用户可批准或要求修改。

**Why this priority**: 这是 ScriptMind 最核心的入口功能，将用户的模糊创意转化为可执行的结构化大纲，是整个创作流程的起点。

**Independent Test**: 用户在 Agent 交互面板输入一句话梗概，选择风格预设后点击发送，界面显示 Agent 阶段进度，60 秒内返回完整大纲，用户可审核批准。

**Acceptance Scenarios**:

1. **Given** 用户在 Agent 交互面板输入一句话梗概并选择风格预设，**When** 点击发送，**Then** 界面显示 Agent 阶段进度标签（Showrunner → WorldBuilder → CharacterDesigner → ScriptDoctor），实时流式输出当前 Agent 的文本内容
2. **Given** 大纲生成完成，**When** 系统进入人类审核节点，**Then** 界面显示大纲内容预览和审核操作按钮（批准 / 要求修改 / 手动编辑）
3. **Given** 用户在审核节点点击"批准"，**When** 审核通过，**Then** 大纲保存成功，大纲查看器显示集数规划和每集摘要
4. **Given** 用户在审核节点点击"要求修改"并输入反馈，**When** 提交反馈，**Then** 系统根据反馈重新生成修订版大纲，界面继续显示 Agent 阶段进度
5. **Given** 用户输入过于简短或模糊（如"写个故事"），**When** 系统尝试解析，**Then** 界面提示用户补充关键信息（题材、目标形态、大致集数）
6. **Given** AI 生成过程中 Token 用量达到警告阈值（80%），**When** 系统检测到，**Then** 界面显示预算警告提示
7. **Given** AI 生成过程中 Token 用量达到硬上限（100%），**When** 系统检测到，**Then** Agent 任务暂停，界面提示用户调整预算

### User Story 3 — 大纲细化为标准剧本 (Priority: P1)

用户在大纲查看器中点击"细化为剧本"，系统将大纲转化为包含场景、动作、对白的标准剧本格式。用户可在剧本编辑器中查看和修改生成的剧本内容。

**Why this priority**: 从大纲到剧本的转化是核心创作流程的关键步骤，是编辑器功能的前提。

**Independent Test**: 用户在已生成大纲的项目中点击"细化为剧本"，系统生成包含场景标题、动作和对白的结构化剧本，用户可在编辑器中查看。

**Acceptance Scenarios**:

1. **Given** 项目已有定稿大纲，**When** 用户点击"细化为剧本"，**Then** 系统启动 Agent 任务，界面显示生成进度
2. **Given** 剧本生成完成，**When** 用户进入剧本编辑器，**Then** 编辑器显示按场景分割的结构化剧本内容
3. **Given** 剧本生成过程中出现错误，**When** 系统检测到错误，**Then** 界面显示错误信息和重试选项

---

### User Story 4 — 专业剧本编辑器 (Priority: P1)

用户在专业剧本编辑器中编写和修改剧本。编辑器支持六种元素类型（场景标题、动作、角色、对白、括号说明、转场），通过 Tab 键快速切换元素类型，左侧边栏显示场景导航，场景编号自动更新。

**Why this priority**: 剧本编辑器是用户直接操作剧本的核心界面，其易用性直接影响创作效率和用户体验。

**Independent Test**: 用户在编辑器中通过 Tab 键切换元素类型，输入对白内容，场景编号自动显示和更新。

**Acceptance Scenarios**:

1. **Given** 用户在编辑器中输入内容，**When** 按下 Tab 键，**Then** 当前元素类型按 scene_heading → action → character → dialogue → parenthetical → transition 顺序循环切换
2. **Given** 用户在对白元素后按 Enter，**When** 新段落创建，**Then** 默认元素类型为 action
3. **Given** 用户在角色元素后按 Enter，**When** 新段落创建，**Then** 默认元素类型为 dialogue
4. **Given** 用户在空内容元素上按 Backspace，**When** 元素被删除，**Then** 焦点自动移到上一个元素
5. **Given** 编辑器中有多个场景标题，**When** 用户编辑内容导致场景顺序变化，**Then** 左侧边栏的场景编号自动更新
6. **Given** 用户选中一段内容，**When** 点击"AI 优化建议"，**Then** 系统返回 2-3 种不同风格的改写方案供选择
7. **Given** 用户修改剧本内容，**When** 修改完成，**Then** 系统以 30 秒防抖间隔自动保存（防丢数据），用户手动保存时创建版本快照

### User Story 5 — 角色管理面板 (Priority: P2)

用户在角色面板中查看和编辑项目中的角色档案。角色列表按角色类型分组显示（主角、反派、配角、次要角色）。用户可查看角色详情（性格、外貌、背景、目标、关系、对白风格、弧光）并手动编辑。

**Why this priority**: 角色管理是创作过程中的重要辅助功能，帮助用户维护角色一致性和关系连贯性。

**Independent Test**: 用户在角色面板中查看角色列表，点击某个角色查看详情，修改角色描述后保存成功。

**Acceptance Scenarios**:

1. **Given** 项目中已有 AI 生成的角色列表，**When** 用户打开角色面板，**Then** 显示按角色类型分组的角色列表
2. **Given** 用户点击某个角色卡片，**When** 角色详情展开，**Then** 显示角色的完整信息（描述、性格、外貌、背景、目标、关系、对白风格、弧光）
3. **Given** 用户修改角色的某个属性（如描述），**When** 点击保存，**Then** 角色信息更新成功
4. **Given** 角色面板中无角色数据，**When** 用户打开面板，**Then** 显示空状态提示，引导用户通过 AI 生成或手动添加

---

### User Story 6 — 伏笔追踪面板 (Priority: P2)

用户在伏笔追踪面板中查看项目中所有伏笔的状态。面板按状态分组显示（已埋设 / 已回收），每个伏笔显示埋设内容、回收内容、关联角色、重要性和提示集数。用户可手动标记伏笔回收状态。

**Why this priority**: 伏笔追踪是区别于简单文本编辑器的关键能力，帮助用户维护多集连续剧的连贯性。

**Independent Test**: 用户在伏笔面板中查看伏笔列表，将一个"已埋设"的伏笔标记为"已回收"并填写回收内容，状态更新成功。

**Acceptance Scenarios**:

1. **Given** 项目中已有伏笔记录，**When** 用户打开伏笔追踪面板，**Then** 显示按状态分组的伏笔列表（已埋设 / 已回收）
2. **Given** 用户点击某个伏笔的"标记回收"按钮，**When** 填写回收内容并提交，**Then** 伏笔状态更新为"已回收"
3. **Given** 伏笔列表中有高重要性的未回收伏笔，**When** 用户查看面板，**Then** 高重要性伏笔有视觉突出标记
4. **Given** 伏笔面板中无伏笔数据，**When** 用户打开面板，**Then** 显示空状态提示

---

### User Story 7 — 版本控制与历史回溯 (Priority: P3)

用户在版本历史面板中查看剧本的版本历史列表。可选择任意版本查看内容，进行版本回溯，或对比两个版本的差异。

**Why this priority**: 创作过程需要反复迭代，版本控制保障用户可以安全地尝试不同方向而不丢失工作成果。

**Independent Test**: 用户进行多次编辑后，在版本历史中选择一个早期版本进行回溯，确认内容恢复正确。

**Acceptance Scenarios**:

1. **Given** 剧本有多次编辑记录，**When** 用户打开版本历史面板，**Then** 显示版本列表（按时间倒序），每条显示版本号、时间和变更摘要
2. **Given** 用户选择一个早期版本，**When** 点击"回溯到此版本"，**Then** 剧本内容恢复到该版本状态，当前版本作为新快照保留
3. **Given** 用户选择两个版本，**When** 点击"对比差异"，**Then** 以高亮方式展示两个版本之间的增删改差异
4. **Given** 版本历史列表为空，**When** 用户打开面板，**Then** 显示空状态提示

---

### User Story 8 — 质量评估仪表盘 (Priority: P3)

用户在质量评估仪表盘中查看剧本的质量指标。仪表盘展示钩子强度、节奏曲线、角色均衡、对白占比、场景多样性等指标，以及伏笔回收状态和总体评分。

**Why this priority**: 质量评估帮助用户量化剧本质量，辅助创作决策，但不阻塞核心创作流程。

**Independent Test**: 用户在质量仪表盘中查看各指标数值和状态，指标在编辑保存后 3 秒内更新。

**Acceptance Scenarios**:

1. **Given** 项目有剧本内容，**When** 用户打开质量评估仪表盘，**Then** 显示各项质量指标的数值、目标范围和状态（通过 / 警告）
2. **Given** 质量指标中有警告项（如未回收伏笔），**When** 用户查看仪表盘，**Then** 警告项有视觉突出标记和详细说明
3. **Given** 用户修改剧本后保存，**When** 仪表盘刷新，**Then** 指标在 3 秒内更新为最新数值

---

### User Story 9 — 全局设置 (Priority: P3)

用户在设置页面配置 API Key（支持多个 LLM 提供商）和查看可用模型列表。设置页面显示已配置的 API Key（脱敏显示）和配置状态。

**Why this priority**: API Key 配置是系统运行的前提，但只需配置一次，优先级低于核心创作功能。

**Independent Test**: 用户在设置页面添加一个 API Key，保存后显示配置成功，Key 以脱敏形式显示。

**Acceptance Scenarios**:

1. **Given** 用户访问设置页面，**When** 页面加载，**Then** 显示已配置的 API Key 列表（脱敏显示 provider 和 key_preview）和可用模型列表
2. **Given** 用户输入新的 API Key 并选择提供商，**When** 点击保存，**Then** API Key 配置成功，列表中显示脱敏的 key_preview
3. **Given** 用户配置了 API Key，**When** 查看可用模型列表，**Then** 对应提供商的模型显示为"已配置"状态

---

### User Story 10 — 文件/URL 导入 (Priority: P2)

用户在导入页面上传文件或输入 URL，系统自动解析并转换为剧本格式。支持 `.txt`、`.docx`、`.md`、`.fountain` 四种文件格式和公开网页 URL。

**Why this priority**: 文件导入扩展了用户群，让已有作品的创作者能快速使用系统，但不阻塞核心创作流程。

**Independent Test**: 用户上传一个 `.txt` 文件，系统解析后生成剧本，用户可在编辑器中查看。

**Acceptance Scenarios**:

1. **Given** 用户在导入页面选择上传文件，**When** 选择一个 `.txt` 文件并点击"导入"，**Then** 系统显示导入进度，完成后展示转换结果
2. **Given** 用户在导入页面输入 URL，**When** 点击"抓取并改编"，**Then** 系统显示抓取进度，完成后展示转换结果
3. **Given** 上传的文件超过 50 万字，**When** 系统检测到，**Then** 提示文件过大并建议分批导入
4. **Given** 输入的 URL 无法访问，**When** 系统尝试抓取，**Then** 提示 URL 无效并建议手动输入

### Edge Cases

- 用户在 Agent 生成过程中刷新页面或导航离开，如何处理？→ 生成任务在后端继续执行，导航栏显示小型进度指示器（旋转图标 + 阶段名称），用户返回后通过 WebSocket 重新连接查看进度
- 用户在编辑器中连续快速按 Tab 切换元素类型，如何保证响应流畅？→ 前端即时切换，不依赖后端响应
- WebSocket 连接中断，如何处理？→ 自动重连（指数退避），重连期间显示连接状态提示
- 项目数据量很大（100 集剧本），页面加载是否流畅？→ 虚拟滚动和懒加载，仅渲染可视区域内容
- 用户在多个浏览器标签页同时编辑同一剧本，如何处理？→ 最后写入保存策略，版本历史提供恢复能力
- LLM API 返回 429 速率限制错误，前端如何展示？→ 显示重试进度提示，告知用户系统正在自动重试
- 用户网络不稳定，API 请求超时，如何处理？→ 显示超时提示和重试按钮，保存草稿状态
- 后端服务完全不可用（Docker 容器停止），如何处理？→ 显示全屏错误页，提供"重试连接"按钮，已编辑草稿保留在本地存储中

## Requirements *(mandatory)*

### Functional Requirements

**页面与导航**

- **FR-001**: 系统 MUST 提供首页，展示平台介绍、工作流程说明、Agent 团队介绍和核心能力描述
- **FR-002**: 系统 MUST 提供项目列表页，展示所有已创建项目（按更新时间倒序），支持新建项目和删除项目（二次确认对话框，要求用户输入项目名称确认删除，级联删除所有关联数据）
- **FR-003**: 系统 MUST 提供项目工作台页面，作为 ScriptMind 模块的主入口，包含 Agent 交互面板、大纲查看器和模块导航
- **FR-004**: 系统 MUST 提供剧本编辑器页面，包含编辑器主体、场景导航侧边栏、角色面板和伏笔追踪面板
- **FR-005**: 系统 MUST 提供大纲生成页面（/outline），包含创意输入区、风格预设选择和大纲查看器
- **FR-006**: 系统 MUST 提供文件导入页面（/import），包含文件上传区和 URL 输入区
- **FR-007**: 系统 MUST 提供全局设置页面，包含 API Key 管理和模型列表
- **FR-008**: 系统 MUST 提供全局导航栏，包含项目名称、Pipeline 阶段导航和设置入口
- **FR-008b**: 全局导航栏 MUST 在 Agent 任务执行期间显示小型进度指示器（旋转图标 + 当前阶段名称），即使用户导航到其他页面也能看到
- **FR-009**: 系统 MUST 为空状态提供引导提示，帮助用户理解下一步操作

**Agent 交互**

- **FR-010**: Agent 交互面板 MUST 支持用户输入文本并发送给 Agent 处理
- **FR-011**: Agent 交互面板 MUST 在 Agent 任务执行期间实时显示当前 Agent 阶段标签（Showrunner / WorldBuilder / CharacterDesigner / ScriptDoctor）
- **FR-012**: Agent 交互面板 MUST 实时流式输出当前 Agent 的文本内容，无需等待全部完成
- **FR-013**: Agent 交互面板 MUST 在人类审核节点显示大纲内容预览和审核操作按钮（批准 / 要求修改 / 手动编辑）
- **FR-014**: Agent 交互面板 MUST 支持用户输入修改反馈并触发重新生成
- **FR-015**: Agent 交互面板 MUST 在 Token 用量达到警告阈值时显示预算警告提示
- **FR-016**: Agent 交互面板 MUST 在 Token 用量达到硬上限时暂停交互并提示用户调整预算
- **FR-017**: Agent 交互面板 MUST 在 LLM API 返回 429 错误时显示自动重试进度
- **FR-018**: Agent 交互面板 MUST 显示 Agent 会话的 Token 消耗量

**剧本编辑器**

- **FR-019**: 编辑器 MUST 支持六种元素类型：场景标题 (scene_heading)、动作 (action)、角色 (character)、对白 (dialogue)、括号说明 (parenthetical)、转场 (transition)
- **FR-020**: 编辑器 MUST 支持 Tab 键循环切换元素类型，Shift+Tab 反向切换
- **FR-021**: 编辑器 MUST 在对白后按 Enter 默认创建 action 元素，在角色后按 Enter 默认创建 dialogue 元素
- **FR-022**: 编辑器 MUST 在空内容元素上按 Backspace 删除该元素并将焦点移到上一个元素
- **FR-023**: 编辑器 MUST 在左侧边栏自动显示场景编号，随编辑自动更新
- **FR-024**: 编辑器 MUST 支持选中内容后触发 AI 优化建议，返回 2-3 种改写方案供选择
- **FR-025**: 编辑器 MUST 在用户修改内容后以 30 秒防抖间隔自动保存（防止数据丢失），仅在用户手动保存时创建版本快照
- **FR-026**: 编辑器 MUST 为不同元素类型提供视觉区分（如场景标题加粗、角色名大写、对白缩进等）

**大纲查看器**

- **FR-027**: 大纲查看器 MUST 以结构化方式展示大纲内容，包含集数规划、每集摘要、关键事件和结尾钩子
- **FR-028**: 大纲查看器 MUST 支持逐集展开/折叠查看详情
- **FR-029**: 大纲查看器 MUST 提供"细化为剧本"操作按钮

**角色面板**

- **FR-030**: 角色面板 MUST 按角色类型分组显示角色列表（主角 / 反派 / 配角 / 次要角色）
- **FR-031**: 角色面板 MUST 支持展开角色卡片查看完整信息（描述、性格、外貌、背景、目标、关系、对白风格、弧光）
- **FR-032**: 角色面板 MUST 支持手动编辑角色属性并保存

**伏笔追踪面板**

- **FR-033**: 伏笔追踪面板 MUST 按状态分组显示伏笔列表（已埋设 / 已回收）
- **FR-034**: 伏笔追踪面板 MUST 显示每个伏笔的埋设内容、回收内容、关联角色、重要性和提示集数
- **FR-035**: 伏笔追踪面板 MUST 支持手动标记伏笔回收状态并填写回收内容
- **FR-036**: 伏笔追踪面板 MUST 对高重要性的未回收伏笔提供视觉突出标记

**版本历史**

- **FR-037**: 版本历史面板 MUST 显示版本列表（按时间倒序），每条包含版本号、时间和变更摘要
- **FR-038**: 版本历史面板 MUST 支持选择任意版本进行回溯，回溯时保留当前版本为新快照
- **FR-039**: 版本历史面板 MUST 支持选择两个版本进行 diff 对比，高亮展示增删改差异

**质量评估**

- **FR-040**: 质量评估仪表盘 MUST 展示钩子强度、节奏曲线、角色均衡、对白占比、场景多样性指标
- **FR-041**: 质量评估仪表盘 MUST 显示伏笔回收状态（总数 / 已回收 / 未回收）
- **FR-042**: 质量评估仪表盘 MUST 显示总体评分和各项指标的状态（通过 / 警告）
- **FR-043**: 质量评估仪表盘 MUST 在剧本保存后 3 秒内更新指标

**设置**

- **FR-044**: 设置页面 MUST 显示已配置的 API Key 列表（脱敏显示 provider 和 key_preview）
- **FR-045**: 设置页面 MUST 支持添加新的 API Key（选择提供商 + 输入 Key）
- **FR-046**: 设置页面 MUST 显示可用模型列表及配置状态

**实时通信**

- **FR-047**: 系统 MUST 通过 WebSocket 连接接收 Agent 任务的实时进度更新
- **FR-048**: 系统 MUST 在 WebSocket 连接中断时自动重连（指数退避策略）
- **FR-049**: 系统 MUST 在 WebSocket 重连期间显示连接状态提示
- **FR-049b**: 系统 MUST 在后端服务完全不可达时显示全屏错误页，提供"重试连接"按钮，并将已编辑的草稿保留在本地存储中，后端恢复后可继续工作

**状态管理**

- **FR-050**: 系统 MUST 管理项目列表状态，支持增删改查操作的本地状态同步
- **FR-051**: 系统 MUST 管理当前项目的工作台状态（Agent 会话、大纲、剧本、角色、伏笔）
- **FR-052**: 系统 MUST 管理 Agent 会话状态（阶段、消息列表、Token 消耗、审核状态）
- **FR-053**: 系统 MUST 管理编辑器状态（当前元素类型、光标位置、场景列表）

### Key Entities

- **Project (项目)**: 用户创建的剧本项目，包含标题、题材、目标形态、状态。前端展示为项目卡片，包含标题、题材标签和最后更新时间。
- **Script (剧本)**: 项目的核心内容，包含完整的剧集结构。前端在编辑器中以结构化方式展示。
- **Outline (大纲)**: AI 生成的结构化大纲，包含集数规划、每集摘要和钩子。前端在大纲查看器中展示。
- **Character (角色)**: 角色档案，包含名称、类型、描述、性格等。前端在角色面板中以卡片形式展示。
- **Foreshadow (伏笔)**: 伏笔记录，包含埋设内容、回收状态、重要性。前端在伏笔追踪面板中分组展示。
- **ScriptVersion (版本)**: 剧本的版本快照。前端在版本历史面板中以列表展示。
- **QualityMetrics (质量指标)**: 剧本质量评估结果。前端在质量仪表盘中以图表展示。
- **AgentSession (Agent 会话)**: Agent 任务的执行记录。前端在 Agent 交互面板中展示进度和消息。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从首页进入项目列表页的操作在 2 秒内完成页面加载
- **SC-002**: 项目列表页在 50 个项目以内时，页面加载时间不超过 1 秒
- **SC-003**: 用户输入一句话梗概后，Agent 交互面板在 1 秒内显示阶段进度标签
- **SC-004**: Agent 流式输出内容的显示延迟不超过 500 毫秒
- **SC-005**: 剧本编辑器的元素类型切换响应时间低于 50 毫秒，用户无感知延迟
- **SC-006**: 编辑器在 100 集剧本（每集 20 个场景）规模下保持流畅交互（帧率不低于 30fps）
- **SC-007**: 版本历史列表在 100 个版本以内时，加载时间不超过 1 秒
- **SC-008**: 版本回溯操作在 5 秒内完成
- **SC-009**: 质量评估仪表盘在剧本保存后 3 秒内完成指标更新
- **SC-010**: WebSocket 断线后自动重连成功率达到 95% 以上
- **SC-011**: 所有页面在移动端（375px 宽度）下可用，核心功能不丢失
- **SC-012**: 用户首次使用时，90% 能在 5 分钟内完成"创建项目 → 生成大纲 → 查看大纲"的完整流程

## Assumptions

- 后端 API 契约完全遵循 001-scriptmind-screenplay-factory 的 API Contracts，前端不做任何 API 格式的假设或修改
- 前端技术栈保持 Next.js 14 + shadcn/ui + Tailwind CSS + Zustand + Slate.js，不引入新的核心依赖
- 初始版本为单用户本地部署，无需认证机制
- 前端通过 WebSocket 接收 Agent 任务的实时进度更新，WebSocket 端点为 `ws://localhost:8080/ws/agent/{session_id}`
- 前端通过 REST API 与后端交互，Base URL 为 `http://localhost:8080/api/v1`
- 剧本编辑器的交互完全在前端完成，不依赖后端实时响应
- 文件导入和 URL 抓取由后端处理，前端仅负责文件上传和结果展示
- 本 Spec 仅覆盖前端部分，后端实现参照 001 spec 的 plan.md
- 现有前端代码将被完全清除，从零重建，不保留任何遗留代码
